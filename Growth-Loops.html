One-liner: A growth loop is a closed system where the output of one action feeds back as input for the next cycle. Unlike funnels (linear, leak), loops compound over time. Popularized by Reforge (Brian Balfour, Casey Winters).

Loops vs Funnels

Why compounding beats linear

The Problem with Funnels

AARRR is a funnel: users enter at the top, some leak out at each stage. To grow, you pour more in. It's linear — double the input, double the output (at best). Funnels don't explain how companies like Instagram, Slack, or Pinterest grew exponentially.

Growth Loops explain compounding. The output of one cycle becomes the input of the next. Each cycle is bigger than the last. 10 users create content that attracts 15, who create content that attracts 22, and so on.

📉
Funnel (Linear)
100 → 50 → 25 → 12 → 6
Each stage loses users. Growth = pour more in.
🔄
Loop (Compounding)
10 → 15 → 22 → 33 → 50
Each cycle's output feeds the next. Growth = self-reinforcing.

🧠 Memory Trick

Funnel = waterfall (water goes one direction, downhill). Loop = flywheel (each push makes the next push easier). Amazon's flywheel: lower prices → more customers → more sellers → better selection → lower prices...

3 Types of Growth Loops

Viral, Content, and Paid

📢 1. Viral Loop

Users invite or attract other users as a natural byproduct of using the product.

User signs up → Invites friends → Friends sign up → They invite friends → 🔄
Examples: WhatsApp (need contacts to use), Dropbox (share files = invite), Slack (invite team to collaborate). Key metric: Viral coefficient (K). K > 1 = exponential growth.
📝 2. Content Loop

Users create content that gets indexed/shared, attracting new users who create more content.

User creates content → Content indexed by Google/social → New user discovers it → Signs up → Creates content → 🔄
Examples: Pinterest (pin images → Google indexes → new users find pins), Quora, Stack Overflow, YouTube. Key metric: Content created per user, SEO impressions.
💰 3. Paid Loop

Revenue from users funds acquisition of more users, as long as LTV > CAC.

Acquire user (paid) → User pays subscription → Revenue funds more ads → Acquire more users → 🔄
Examples: Netflix, most SaaS, Spotify. Key metric: LTV:CAC ratio & payback period. Limitation: Only compounds if LTV grows faster than CAC (which gets harder at scale).

⚠️ Not All Loops Are Equal

Viral loops are the most powerful (free, exponential). Content loops are durable but slow to build. Paid loops are the most common but the weakest — they stop when you stop spending. The best companies have multiple loops reinforcing each other.

Loop Mechanics

What makes a loop actually work

The 3 Parts of Every Loop

1
Input
What enters the loop. New user, new content, new revenue. For the first cycle, this is "seeded" externally (marketing, PR, founder hustle).
2
Action
What the user does inside the product that creates value for others. Shares, creates, invites, reviews, publishes.
3
Output → New Input
The result of that action generates new input: new users discover, new content attracts, revenue funds more acquisition. This is the feedback mechanism.

📊 Measuring Loop Health

🔄 Cycle Time
How long does one full loop take? Instagram: minutes (post → seen → follow). Airbnb: weeks (stay → review → new booking). Shorter = faster compounding.
📈 Conversion Rate
What % of output converts back to input? If 100 users create content that brings 30 new users, conversion = 30%. Higher = stronger compounding.
🌊 Amplitude
How much output per input? If each user invites 3 people on average, amplitude = 3. But if only 20% accept, effective amplitude = 0.6.
🛡️ Defensibility
Can competitors copy this loop? Content loops build SEO moats. Network effects build user moats. Paid loops have zero defensibility.

💡 The Compounding Formula

Loop strength = Amplitude × Conversion ÷ Cycle Time. To grow faster: increase how many people each user reaches (amplitude), improve % who convert (conversion), or shorten the time for one full cycle. Most teams only think about amplitude. The real leverage is often in reducing cycle time.

Real Example

Growth loops for Skoove / Trala

🎹 Skoove's Current & Potential Loops

💰 Active Loop: Paid Acquisition
User subscribes ($9.99/mo) → Revenue funds App Store ads → New user installs → Some subscribe → 🔄
Strength: Weak. Stops when you stop spending. CAC rising in music app category.
📢 Potential Loop: Social Proof
User hits milestone (first song!) → App prompts "Share your progress" → Posts video on Instagram/TikTok → Friends see and think "I could do that" → Download app → 🔄
Strength: Strong if activated. Free, authentic, emotional trigger. But requires investment in share UX.
📝 Potential Loop: Content/SEO
User learns "Für Elise" → Skoove creates blog post "How to play Für Elise on piano" → Ranks on Google → Beginner finds it → Downloads app → Learns another song → More content created → 🔄
Strength: Slow but durable. Builds SEO moat over time. Low marginal cost.

🔍 Strategic Insight

Skoove is running the weakest loop (paid) as its primary growth engine. The social proof loop is the biggest untapped opportunity — music learning has a natural "look what I can do!" share trigger that most apps under-exploit. Investing in beautiful share UX + milestone moments could shift from paid to viral growth.

When to Use

Right situations and wrong ones

✅ Use Growth Loops When...

  • ✔️ Designing sustainable growth strategy
  • ✔️ Current growth depends too heavily on paid ads
  • ✔️ Product has natural share/invite triggers
  • ✔️ PM interview asks "how would you grow this product?"
  • ✔️ Building a growth model or financial projections

⛔ Skip Growth Loops When...

  • ❌ Pre-product-market-fit (loops amplify what exists — good or bad)
  • ❌ Retention is broken (loops on leaky products = faster waste)
  • ❌ Need to diagnose WHERE growth is stuck (use AARRR first)
  • ❌ B2B enterprise with manual sales
  • ❌ Feature prioritization (use RICE/ICE)
